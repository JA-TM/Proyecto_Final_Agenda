## 1) Clase `Configuracion`

Es un objeto que guarda los 4 datos del `config.txt`. Se crea leyendo el config
(ver más abajo) y se consulta con sus *getters*.

| Método | Qué devuelve | Ejemplo |
|---|---|---|
| `getAnio()` | el año (int) | `2025` |
| `getMes()` | el mes (int) | `11` |
| `getIdiomaEntrada()` | idioma de entrada (String) | `"esp"` |
| `getIdiomaSalida()` | idioma de salida (String) | `"ENG"` |

---

## 2) Leer el `config.txt`

Método **estático** (se llama directamente, sin crear objeto):

```java
Configuracion config = GestionDatosTiempo.leerConfig("config.txt");

int anio = config.getAnio();              // 2025
int mes = config.getMes();                // 11
String idiomaEntrada = config.getIdiomaEntrada();  // "esp"
String idiomaSalida = config.getIdiomaSalida();    // "ENG"
```

## 3) Diccionario de idiomas (traducciones)

Métodos **de objeto** (hay que crear un `gestor` primero). Sirve para pedir
cualquier palabra por su código, en el idioma que se cargue.

```java
GestionDatosTiempo gestor = new GestionDatosTiempo();

// Cargar el idioma que se quiera (ESP para la entrada, ENG para la salida)
gestor.cargarDiccionario("ENG");

String titulo = gestor.getTraduccion("001");   // "Schedule"
String cerrado = gestor.getTraduccion("007");  // "Closed"
```

### Códigos disponibles en los diccionarios

| Código | Significado | ESP | ENG |
|---|---|---|---|
| 001 | Título | Agenda | Schedule |
| 002 | Nombres de los días | Lunes,Martes,... | Monday,Tuesday,... |
| 003 | Letras de la máscara de días | LMXJVSD | MTWHFSN |
| 004 | Nombres de los meses | Enero,Febrero,... | January,February,... |
| 005 | Etiquetas | Año,Mes,Semana,Día | Year,Month,Week,Day |
| 006 | "Generado por" | Generado por | Generated by |
| 007 | Actividad "Cerrado" | Cerrado | Closed |
| 008 | Error | Error | Error |

> Nota: para leer la ENTRADA (peticiones) se usa `internacional.ESP`;
> para escribir la SALIDA (HTML) se usa `internacional.ENG`.

## 4) Herramientas de fechas

Métodos **estáticos**. Todos reciben los datos en el orden **(año, mes, día)**.

```java
// Cuántos días tiene el mes (28, 30, 31)
int dias = GestionDatosTiempo.getDiasDelMes(2025, 11);          // 30

// Qué día de la semana es una fecha (1=Lunes ... 7=Domingo)
int diaSem = GestionDatosTiempo.getDiaDeLaSemana(2025, 11, 1);  // 6 (sábado)

// En qué fila/semana del calendario cae un día (0 = primera fila)
int fila = GestionDatosTiempo.getFilaSemanaMes(2025, 11, 30);   // 4
```

### Cómo interpretar `getDiaDeLaSemana`

| Nº | Día |
|---|---|
| 1 | Lunes |
| 2 | Martes |
| 3 | Miércoles |
| 4 | Jueves |
| 5 | Viernes |
| 6 | Sábado |
| 7 | Domingo |

### Para qué sirve cada una

- `getDiasDelMes` → saber hasta qué número llega el mes.
- `getDiaDeLaSemana` → saber en qué **columna** del calendario va un día.
- `getFilaSemanaMes` → saber en qué **fila/semana** del calendario va un día.

Con la columna (día de la semana) y la fila (semana del mes) se sabe la casilla
exacta donde colocar cada día en la tabla HTML.

## ⚠️ IMPORTANTE para la integración (leer antes de juntar las partes)

La clase `GestionDatosTiempo` tiene ahora un método `main` que es **solo de
prueba** (sirve para comprobar que este módulo funciona por separado).

Cuando montemos el `Main.java` definitivo del equipo, **habrá que apartar este
`main` de prueba**, porque un proyecto no puede tener dos `main` "de verdad"
compitiendo y daría error o confusión.

**Qué hacer en la integración:** comentar el `main` de prueba de esta clase
(ponerle `/*` antes y `*/` después). Así se conserva por si hay que volver a
probar el módulo, pero Java lo ignora y no estorba.

```java
/*  *** Prueba (comentada para la integracion) ***
public static void main(String[] args) {
    ... (todo el main de prueba) ...
}
*/
```

El `main` que manda el programa completo será el de `Main.java`, que llamará a
las funciones de este módulo (leerConfig, cargarDiccionario, getTraduccion y las
de fecha) en el orden correcto.

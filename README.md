# Proyecto g_agenda — Grupo ROSIJO

Aplicación **Java** que genera la agenda de ocupación de espacios (salas) a partir
de unas peticiones leídas desde ficheros de texto. Como resultado produce un
fichero **HTML por cada sala** y un fichero de **incidencias** con los conflictos
detectados.

Proyecto final del módulo · Entrega: **20 de julio de 2026**

---

## Descripción

El programa lee la configuración y las peticiones de reserva desde ficheros de
texto, procesa las solicitudes resolviendo los conflictos de horario según unas
reglas de prioridad, y genera la agenda mensual en formato web.

**Todo entra y sale por ficheros.** El programa no pide datos por pantalla ni
muestra resultados por consola (salvo los `main` de prueba durante el desarrollo).

### Flujo del programa

```
config.txt + peticions.txt + internacional.ESP/ENG
                  │
                  ▼
         [ Motor Java: matriz de ocupación ]
                  │
                  ▼
      Sala1.html + Sala2.html + incidencias.log
```

## Ficheros de entrada

### `config.txt`
```
2025 11
esp ENG
```
- Línea 1: **año** y **mes** a procesar.
- Línea 2: **idioma de entrada** e **idioma de salida**.

### `peticions.txt`
Cada línea tiene 6 campos separados por espacios:
```
ReunioJava Sala1 25/09/2025 02/11/2025 LMXJV 08-10
     │       │        │          │       │     │
 actividad espacio  f.inicio   f.fin   días  horas
```
- **Máscara de días:** `L`unes, `M`artes, mi`X`rcoles, `J`ueves, `V`iernes, `S`ábado, `D`omingo.
- **Máscara de horas:** franjas separadas por `_`, cada franja con `-`.
  Ej: `08-10_19-21` → horas 8, 9, 19 y 20. Máximo 5 franjas.

### `internacional.ESP` / `internacional.ENG`
Diccionarios de traducción, formato `codigo;palabra`:

| Código | Significado | ESP | ENG |
|---|---|---|---|
| 001 | Título | Agenda | Schedule |
| 002 | Días | Lunes,Martes,… | Monday,Tuesday,… |
| 003 | Letras máscara | LMXJVSD | MTWHFSN |
| 004 | Meses | Enero,Febrero,… | January,February,… |
| 005 | Etiquetas | Año,Mes,Semana,Día | Year,Month,Week,Day |
| 006 | Generado por | Generado por | Generated by |
| 007 | Actividad cerrada | Cerrado | Closed |
| 008 | Error | Error | Error |

## Reglas de negocio

1. **`Cerrado` tiene prioridad absoluta.** Se procesa siempre primero,
   independientemente de su posición en el fichero. Se pinta en gris (`#b2aaaa`).
2. **Las actividades se asignan por orden de aparición.** Si una actividad se
   repite en varias líneas, sus peticiones se **acumulan** y conservan la
   prioridad de su primera aparición.
3. **En caso de conflicto, gana la primera actividad asignada.** Las horas
   rechazadas se registran en `incidencias.log`.
4. **Las peticiones inválidas se descartan** y se anotan en el log:
   - Fecha de fin anterior a la de inicio (ej: `01/06/2025 → 02/02/2025`).
   - Franjas horarias mal formadas (`08-08`, `23-02`).

## Decisiones del grupo

- **Idiomas: español (entrada) e inglés (salida).** Se descartó el catalán para
  evitar incidencias de codificación y de nombres.
- El `peticions.txt` original del profesor estaba en catalán. Fue **traducido al
  español** por el grupo: `Tancat` → `Cerrado`, y en la máscara de días
  miércoles `C` → `X` y domingo `G` → `D`.
- Se procesan **dos salas** (Sala1 y Sala2), generando un HTML para cada una.

## Cómo compilar y ejecutar

Desde la carpeta raíz del proyecto:

```bash
# Compilar (deja los .class en la carpeta bin)
javac -d bin src\Main.java src\GestionDatosTiempo.java src\Configuracion.java

# Ejecutar
java -cp bin Main
```

> En Linux/Mac, sustituir `\` por `/` en las rutas.

El resultado se genera en la raíz: `Sala1.html`, `Sala2.html` e `incidencias.log`.
